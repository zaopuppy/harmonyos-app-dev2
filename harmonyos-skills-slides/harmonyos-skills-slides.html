ArkTS 语法差异

大模型已经知道这些差异。通过编译告警可以直接修复。

严格类型

禁用 any，禁止动态类型

不可变更新

状态修改必须创建新对象

完整标注

显式声明所有类型

// TypeScript 特性在 ArkTS 中不允许 let data: any; // ✗ 错误！禁用 any 类型 let value: unknown; (value as string).length; // ✗ 错误！unknown 断言 let obj = {}; obj['key'] = value; // ✗ 错误！动态属性访问 class A { x: number = 0; } class B { x: number = 0; } let a: A = new B(); // ✗ 错误！结构化类型 type T = typeof someVar; // ✗ 错误！typeof 用于类型 type K = keyof SomeType; // ✗ 错误！keyof 操作符
✓ 策略：提供少样本示例，主要通过编译错误指导解决

ArkUI 组件语法

ArkUI 组件语法是一套 DSL，和 ArkTS 完全是两套语法。大部分时候可以正常生成。

@Component struct AnimatedButton { @State scale: number = 1; @State opacity: number = 1; build() { Button('Animated') .scale({ x: this.scale, y: this.scale }) .opacity(this.opacity) .animation({ duration: 300, curve: Curve.EaseInOut }) .onTouch((event: TouchEvent) => { if (event.type === TouchType.Down) { this.scale = 0.95; this.opacity = 0.8; } else if (event.type === TouchType.Up) { this.scale = 1; this.opacity = 1; } }) } }
✓ 策略：提供少样本示例，主要通过编译错误指导解决

SDK 索引方案

轻量 SDK 索引（索引耗时 <1s），提供 API 的实时查询，相当于接入了一个 SDK 知识库。

工作原理

  1. 1. 扫描 SDK 的 .d.ts 类型定义文件
  2. 2. 解析模块路径、类成员、方法签名
  3. 3. 构建可查询的索引数据库
  4. 4. 提供命令行查询接口

索引范围

  • • @ohos.* — 系统 API
  • • @kit.* — ArkUI 套件
  • • @system.* — 系统能力
  • • @hms.* — HMS 套件
python check_sdk_imports.py --query "WebviewController"
@ohos.web.webview.webview.WebviewController @kit.ArkWeb.webview.WebviewController
✓ 策略：轻量 SDK 索引 + 实时 API 查询 + 智能纠错推荐

HDC 开发者工具

轻量自动化验证的基石。辅助自动化验证（编译后安装、分析 crash 等）。目前大模型基本不具备相关知识。

hdc shell

进入设备 shell

hdc install app.hap

安装应用包

hdc file send local remote

推送文件到设备

hdc shell bm dump -a

查看已安装应用

hdc shell pidof com.example

获取进程 PID

hdc shell hilog -r

查看实时日志

✓ 策略：提供常用命令，作为知识库的补充
